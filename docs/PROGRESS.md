## Known deviations / follow-ups

- `npm audit` reports 6 moderate findings in dev-only transitive deps (`drizzle-kit`'s esbuild-kit bundler, Next's internal postcss). See `docs/DECISIONS.md` (2026-07-15 entry). Revisit on next dependency bump.
- `middleware.ts` was renamed to `proxy.ts` per Next.js 16's new convention.
- pg-boss queue `policy` is immutable after creation; changing `QUEUE_CONFIG` policy for an existing queue requires deleting it first (see `docs/DECISIONS.md`).
- e2e coverage for authenticated pages (`workspace-shell.spec.ts`, `storage.spec.ts`) is `test.skip()`-ed — this Clerk instance is OAuth-only, so headless test sign-in isn't set up yet. See `docs/TEST_PLAN.md`. (Manually verified working in a real browser instead — see T1.1.2 above.)
- Clerk's test-mode secret key is now read from a GitHub Actions encrypted secret (`CLERK_SECRET_KEY`), not hardcoded, after an earlier draft of `ci.yml` was caught staging it in plain text.
- `clerkMiddleware`'s `createRouteMatcher` usage in `proxy.ts` triggers a Clerk deprecation warning in dev logs (Clerk is moving toward resource-based auth checks per-route instead of path-matching middleware). Not urgent — revisit if Clerk removes the API.
