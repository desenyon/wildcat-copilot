## Known deviations / follow-ups

- `npm audit` reports 6 moderate findings in dev-only transitive deps (`drizzle-kit`'s esbuild-kit bundler, Next's internal postcss). See `docs/DECISIONS.md` (2026-07-15 entry). Revisit on next dependency bump.
- Workspace shell uses placeholder/hardcoded course data (`WorkspaceShell.tsx`) — courses aren't in the UI yet even though the DB schema exists. Wire up once M1.1/M1.2 course CRUD ships.
- `middleware.ts` was renamed to `proxy.ts` per Next.js 16's new convention.
- pg-boss queue `policy` is immutable after creation; changing `QUEUE_CONFIG` policy for an existing queue requires deleting it first (see `docs/DECISIONS.md`).
- e2e coverage for authenticated pages (`workspace-shell.spec.ts`, `storage.spec.ts`) is `test.skip()`-ed — this Clerk instance is OAuth-only, so headless test sign-in isn't set up yet. See `docs/TEST_PLAN.md`.
- Clerk's test-mode secret key is committed in `.github/workflows/ci.yml` for the e2e job; move to a GitHub Actions encrypted secret when convenient (see `docs/DECISIONS.md`).
